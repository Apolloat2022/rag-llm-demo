## Data Flow

### Ingestion Flow
```
PDF Upload вҶ’ PyMuPDF Parser вҶ’ Table/Text Extraction
    вҶ’ Parent Chunks (512 tokens, full context)
    вҶ’ Child Chunks (128 tokens, for retrieval)
    вҶ’ Embed Child Chunks вҶ’ ChromaDB
    вҶ’ Store Parent Chunks вҶ’ ChromaDB (docstore)
```

### Query Flow
```
User Question вҶ’ LangGraph Agent вҶ’ Router (tool selection)
    в”ңв”Җв–ә RAG Tool: Embed Query вҶ’ ChromaDB similarity search
    в”Ӯ       вҶ’ Retrieve Child Chunks вҶ’ Fetch Parent Chunks
    в”Ӯ       вҶ’ Cross-Encoder Re-rank вҶ’ Top-K contexts
    в”Ӯ       вҶ’ LLM with citations вҶ’ Response + Source Metadata
    в”Ӯ
    в””в”Җв–ә Quote Tool: Extract parameters вҶ’ Python calculation
            вҶ’ Deterministic result вҶ’ LLM formats response
```

## Key Design Decisions

| Decision | Choice | Reason |
|---|---|---|
| Chunking | Parent-Child (512/128 tokens) | Balance retrieval precision with LLM context |
| Re-ranker | Cross-Encoder (ms-marco) | Fixes cosine-similarity ranking errors |
| Agent | LangGraph | Stateful graph, easy to add nodes/edges |
| Math | Python tool, not LLM | LLMs miscalculate; deterministic code doesn't |
| DB | ChromaDB (local) / Pinecone (prod) | Local dev вҶ’ cloud swap via env var |
